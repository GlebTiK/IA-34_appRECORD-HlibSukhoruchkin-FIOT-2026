## Тема, Мета, Місце розташування

**Тема:** «Веб-орієнтований застосунок каталогу цуценят "Puppy Haven". Розширення: створення бази даних у MySQL, підключення Node.js до MySQL, робота з ORM Sequelize.»

**Мета:** прискорити вибір цуценя для користувача за рахунок зручного веб-каталогу з фото та короткими характеристиками і простої форми заявки на візит, щоб швидко переглянути варіанти та одразу залишити запит на зустріч/бронювання.  
Додатково (ЛР №2): створити базу даних у MySQL, виконати SQL-запити (SELECT/INSERT/UPDATE/DELETE), підключити Node.js до MySQL через `mysql2`, використати ORM Sequelize та реалізувати зв’язок One-to-Many між таблицями.

## Структура веб-застосунку

- **Головна сторінка** — шапка з навігацією, hero-блок «цуценя тижня», блок категорій, прев’ю «Про нас», каталог (таблиця), форма заявки, секція «Відгуки» (заготовка), футер з контактами.
- **Про нас** — сторінка з описом проєкту, бізнес-логікою та переліком вимог, оформлена в стилі головної сторінки (шапка + футер).

### Додано у ЛР №2 (MySQL + Sequelize)

- **MySQL БД**: таблиці `puppies` та `visit_requests`.
- **mysql2**: демонстрація SQL CRUD у `scripts/sql_demo.js`.
- **Sequelize**:
  - моделі `Puppy` та `VisitRequest` (`models/`),
  - зв’язок **One-to-Many**: `Puppy.hasMany(VisitRequest)` та `VisitRequest.belongsTo(Puppy)`.
- **REST API для сайту**:
  - `GET /api/puppies` — віддати каталог цуценят з БД.
  - `POST /api/visit_requests` — створити заявку на візит у БД.
  - `GET /api/visit_requests` — переглянути заявки (для тестування).
  - `GET /api/health` — перевірка підключення до БД.

## Сценарій взаємодії (бізнес-логіка)

1. Користувач відкриває сторінку `http://localhost:3000/app/`.
2. Frontend виконує `GET /api/puppies` і заповнює таблицю каталогу та список у формі.
3. Користувач заповнює форму заявки та надсилає її.
4. Frontend відправляє `POST /api/visit_requests` (ім’я, телефон, дата/час, коментар, обране цуценя).
5. Сервер зберігає заявку в MySQL (таблиця `visit_requests`) і повертає відповідь.
6. Користувач отримує повідомлення-підтвердження.

## SQL-запити (mysql2)

Файл: `scripts/sql_demo.js` (працює через `mysql2/promise`).  
Послідовність:
- `SELECT` — читання перших 5 записів з `puppies`.
- `INSERT` — вставка тестового запису.
- `UPDATE` — оновлення тестового запису.
- `DELETE` — видалення тестового запису.

Запуск:
```bash
npm run sql:demo
```

---

## ORM Sequelize (моделі та зв’язок One-to-Many)

Моделі:
- `models/Puppy.js`
- `models/VisitRequest.js`

Зв’язок (One-to-Many):
- `Puppy.hasMany(VisitRequest, { foreignKey: 'puppy_id' })`
- `VisitRequest.belongsTo(Puppy, { foreignKey: 'puppy_id' })`

## Налаштування та запуск

1. Створити БД у MySQL (приклад назви: `web_backend_lab`).
2. Виконати `sql/schema.sql` (створить таблиці).
3. Скопіювати `.env.example` → `.env` та заповнити `DB_*`.
4. Встановити залежності:
```bash
npm install
```
5. (Опціонально) створити таблиці через Sequelize:
```bash
npm run db:sync
```
6. Заповнити початкові дані каталогу:
```bash
npm run db:seed
```
7. Запустити сервер:
```bash
npm start
```

Перевірка:
- `http://localhost:3000/` → `Hello from Node.js server`
- `http://localhost:3000/app/` → сайт Puppy Haven
- `http://localhost:3000/api/health` → статус підключення БД
- `http://localhost:3000/api/puppies` → каталог з БД
