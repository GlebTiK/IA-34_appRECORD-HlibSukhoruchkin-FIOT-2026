## Тема, Мета, Місце розташування

**Тема:** «Веб-орієнтований застосунок каталогу цуценят "Puppy Haven".»

**Мета:** прискорити вибір цуценя для користувача за рахунок зручного веб-каталогу з фото та короткими характеристиками і простої форми заявки на візит, щоб швидко переглянути варіанти та одразу залишити запит на зустріч/бронювання.  
Додатково (ЛР №2): створити базу даних у MySQL для зберігання каталогу цуценят та заявок, підключити Node.js до MySQL через `mysql2`, використати ORM Sequelize та реалізувати зв’язок One-to-Many між таблицями.

## Опис предметного середовища

**Предметна галузь:** веб-каталог цуценят "Puppy Haven". Сайт надає перелік цуценят (у вигляді таблиці), коротку інформацію про проєкт/команду, контакти та форму для подачі заявки на візит.

У ЛР №2 дані каталогу та заявки на візит зберігаються у MySQL, а веб-сторінки отримують каталог через REST API.

---

## Структура веб-застосунку

- **Головна сторінка** — шапка з навігацією, hero-блок «цуценя тижня», блок категорій, прев’ю «Про нас», каталог (таблиця), форма заявки, секція «Відгуки» (заготовка), футер з контактами.
- **Про нас** — сторінка з описом проєкту, бізнес-логікою та переліком вимог, оформлена в стилі головної сторінки (шапка + футер).

### Додано у ЛР №2 (backend + DB)

- **Node.js + Express сервер** (`server.js`) — віддає статичні сторінки з папки `public/` та реалізує REST API.
- **MySQL БД**:
  - `puppies` — каталог цуценят,
  - `visit_requests` — заявки на візит (FK на `puppies`).
- **mysql2** — демонстрація виконання SQL-запитів (SELECT/INSERT/UPDATE/DELETE) у `scripts/sql_demo.js`.
- **Sequelize** — моделі та зв’язки:
  - `models/Puppy.js`,
  - `models/VisitRequest.js`,
  - One-to-Many: `Puppy.hasMany(VisitRequest)` та `VisitRequest.belongsTo(Puppy)`.

## Сценарій взаємодії (бізнес-логіка)

1. Користувач відкриває сайт і переходить до розділу **Каталог** (через меню або прокрутку).
2. Frontend виконує `GET /api/puppies` і заповнює таблицю з цуценятами та список у формі.
3. Переходить до розділу **Заявка на візит** і обирає цуценя зі списку у формі.
4. Заповнює форму (ім’я, телефон, дата/час, коментар) і надсилає заявку.
5. Frontend відправляє `POST /api/visit_requests`.
6. Сервер зберігає заявку в MySQL і повертає підтвердження.
7. Користувач отримує клієнтське повідомлення-підтвердження (на сторінці).
8. Адміністратор зв’язується для уточнення деталей (поза межами сайту, за контактним номером/поштою).

- `puppy_id` (FK → `puppies.id`)
- `visitor_name`
- `phone`
- `visit_datetime`
- `note`
- `status` (`pending/approved/rejected`)

SQL створення таблиць: `sql/schema.sql`.

## REST API

- `GET /api/health` — перевірка підключення до БД.
- `GET /api/puppies` — отримати каталог цуценят з MySQL.
- `POST /api/visit_requests` — створити заявку на візит.
- `GET /api/visit_requests` — перегляд заявок (для тестування).

---

## SQL-запити (mysql2)

Файл: `scripts/sql_demo.js`.

Запити:
- `SELECT` — вибірка записів з `puppies`.
- `INSERT` — вставка тестового запису у `puppies`.
- `UPDATE` — оновлення тестового запису.
- `DELETE` — видалення тестового запису.

Запуск:
```bash
npm run sql:demo
```

## Налаштування та запуск

1. Створити БД у MySQL (наприклад `web_backend_lab`).
2. Виконати `sql/schema.sql` (створить таблиці).
3. Скопіювати `.env.example` → `.env` і заповнити `MYSQL_URL` або поля `DB_*`.
4. Встановити залежності:
```bash
npm install
```
5. Заповнити початкові дані каталогу:
```bash
npm run db:seed
```
6. Запустити сервер:
```bash
npm start
```

Перевірка:
- `http://localhost:3000/` → сайт Puppy Haven
- `http://localhost:3000/api/health` → статус БД
- `http://localhost:3000/api/puppies` → каталог з БД

## Висновки

У лабораторній роботі №2 веб-застосунок "Puppy Haven" розширено збереженням даних у MySQL: створено таблиці `puppies` та `visit_requests` із зв’язком One-to-Many, виконано SQL CRUD-запити через `mysql2`, використано ORM Sequelize для моделей та взаємодії з БД. Каталог цуценят завантажується через REST API, а заявки на візит зберігаються у базі даних.
